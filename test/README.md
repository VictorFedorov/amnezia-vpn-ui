# Интеграционные тесты

Этот каталог содержит скрипты для тестирования консистентности данных между приложением и серверами.

## get_token.py

Скрипт для получения API токена через логин в систему.

### Использование:

```bash
python test/get_token.py --username admin --password yourpassword
```

### Параметры:

- `--username`: Имя пользователя (обязательный)
- `--password`: Пароль (обязательный)
- `--api-url`: Базовый URL API (по умолчанию `http://localhost:8000/api`)

### Пример:

```bash
python test/get_token.py --username admin --password <your_password>

# Вывод:
✅ Успешная авторизация!
🔑 API Token: eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
💡 Используйте этот токен в integration_test.py:
python test/integration_test.py --client-id X --server-id Y --token eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
💾 Токен сохранен в /tmp/api_token.txt
```

## integration_test.py

Скрипт для проверки консистентности данных VPN клиентов между:
- Базой данных приложения (через API)
- Реальными данными на серверах (через SSH)

### Что проверяет скрипт:

- ✅ Public Key клиентов
- ✅ IP адреса (Allowed IPs)
- ✅ Endpoint адреса
- ✅ Статистика передачи данных (с допустимой погрешностью)

### Использование:

```bash
# Базовое использование
python test/integration_test.py --client-id 1 --server-id 1 --token YOUR_API_TOKEN

# С кастомным API URL
python test/integration_test.py --client-id 1 --server-id 1 --token YOUR_API_TOKEN --api-url http://your-server:8000/api
```

### Параметры:

- `--client-id`: ID клиента в базе данных (обязательный)
- `--server-id`: ID сервера в базе данных (обязательный)
- `--token`: API токен для авторизации (обязательный, или установить переменную `API_TOKEN`)
- `--api-url`: Базовый URL API (по умолчанию `http://localhost:8000/api`)

### Как получить API токен:

1. Авторизуйтесь в веб-приложении
2. Откройте Developer Tools браузера (F12)
3. Перейдите во вкладку Network
4. Выполните любое действие в приложении
5. Найдите запрос с заголовком `Authorization: Bearer <token>`
6. Скопируйте токен после `Bearer `

### Пример использования:

```bash
# 1. Создайте клиента через веб-интерфейс
# 2. Получите его ID из URL или API
# 3. Запустите тест
python test/integration_test.py --client-id 5 --server-id 2 --token eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...

# Ожидаемый вывод:
🚀 Начинаем тестирование клиента ID=5 на сервере ID=2
============================================================
1️⃣ Получаем данные через API...
   Клиент: Иван Иванов (ID: 5)
   Найдено конфигураций: 1
   Сервер: Main Server (192.168.1.100:22)
2️⃣ Получаем данные напрямую с сервера...

🔍 Проверяем конфигурацию wireguard (ID: 10)
✅ Данные wireguard пира консистентны

============================================================
🎉 ВСЕ ДАННЫЕ КОНСИСТЕНТНЫ!
```

### Возможные проблемы:

- **SSH подключение**: Убедитесь, что SSH ключи и credentials корректны
- **API токен**: Проверьте срок действия токена
- **Контейнеры**: Убедитесь, что Docker контейнеры работают на сервере
- **Синхронизация**: Данные могут отличаться если недавно была синхронизация

### Структура скрипта:

- `DataConsistencyTester`: Основной класс для тестирования
- Методы для получения данных через API
- Методы для получения данных напрямую с серверов
- Логика сравнения и отчетности
